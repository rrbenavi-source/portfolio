The stakesThe risk

There are two ways to fail.
One is far worse.

A data assistant can return an error —an inconvenience— or return a number that looks flawless and is wrong. The second ends up on a board slide and nudges a decision in the wrong direction.

The thesis

The problem is almost never the model

The generative-BI conversation misses the point: the language model gets better every quarter and cheaper on its own.

What decides whether you can trust the answer is the context —what each metric means, which table is right, which business rule applies—. The model doesn’t supply that. You do, in the semantic layer.

The conceptWhat it actually is

A translation layer

It turns a cryptic production schema into a clean, documented business-friendly interface the AI can reason over.

The key: the logic lives in the view definition, not the model’s head. If net_revenue already carries WHERE refund_flag = 0, every query inherits that rule. The SQL engine enforces it —not the model’s goodwill.
